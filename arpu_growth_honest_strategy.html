THE HARD TRUTH

You Cannot Just Raise Prices

❌ What Will Cause Churn

  • Telling existing customers "same product, new price"
  • Migrating customers to higher tiers for features they already have
  • Breaking grandfathered pricing mid-contract
  • Using "new pricing" as a cover for a price hike on unchanged value
  • Forcing tier migration without showing new value

✅ What Will Actually Work

  • Correcting underpaying customers (getting more value than they pay for)
  • Activating unused features they're already paying for
  • Upselling with ROI proof — "upgrade pays for itself"
  • Winning back 6,846 churned accounts (new revenue, not repricing)
  • Converting 48,695 missed leads (new revenue, not repricing)

⚡ The ₹2,500 ARPU target must come from mix shift — more customers on Growth/Premium — not from charging existing customers more for the same thing.

LEVER 1 · Pricing Correction (Not a Price Hike)

Fix the Pricing Accidents

Some customers are on old grandfathered plans paying ₹500–₹800/month for features worth ₹2,000+. This is a correction, not a hike.

ScenarioCurrent PriceFeatures They UseFair PriceApproach
Using CM + BE + CRM on old plan₹800/moGrowth-level features₹2,000/moShow value received vs price paid. Migrate at renewal.
Using CM + PMS + BE on old plan₹600/moGrowth-level features₹2,000/moSame — value conversation at renewal
Using only CM on old plan₹900/moBasic-level features₹1,000/moNeutral migration — no price change
Paying MORE than new tier₹3,500/moGrowth-level features₹2,000/moImmediate migration — price DROP = goodwill

✅ Rule: Only migrate at renewal. Never mid-contract. Give 60 days notice. CSM handles every conversation personally.

❌ Never frame it as "price increase." Frame it as "we've simplified pricing and here's what you get now."

05 / 12

LEVER 2 · Feature Activation (They're Already Paying For It)

They Pay For It. They Don't Use It.

If a customer is on Growth but has never activated the Booking Engine — that's a CSM failure. Activation = value realization = retention + upsell trigger.

🎯 Booking Engine Activation

  • Pull: how many Growth/Premium customers have BE enabled but 0 direct bookings?
  • CSM calls every one: "I noticed your Booking Engine isn't getting bookings yet. 30-min session and I'll set it up for you."
  • Once active: monthly "You saved ₹[X] in OTA commissions" email
  • This is the #1 retention tool — hotels that use BE never leave
  • Target: 60% activation within 60 days

📊 CRM Activation

  • Pull: how many Growth customers have 0 contacts in CRM?
  • CSM session: "Let me import your guest list and show you how to run a WhatsApp campaign"
  • First campaign result = proof of value = stickiness
  • Hotels using CRM campaigns have 40%+ lower churn (industry data)
  • Target: 50% CRM activation within 90 days

💡 Feature activation doesn't increase ARPU directly — but it dramatically reduces churn, which is the same as increasing revenue. A customer who uses 3+ features churns at 5% vs 30% for single-feature users.
